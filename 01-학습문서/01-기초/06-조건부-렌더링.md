# 조건부 렌더링

조건부 렌더링은 **데이터 상태에 따라 다른 UI를 보여주는 것**입니다. React에서 UI는 데이터의 결과이므로, 조건부 렌더링은 가장 기본적인 패턴 중 하나입니다.

JSX 안에 `if` 문을 바로 쓸 수 없기 때문에 몇 가지 패턴을 익혀야 합니다.

## 패턴 1: `&&` 연산자

조건이 참일 때만 element를 렌더링합니다. "이 조건이면 보여준다"는 상황에 씁니다.

```tsx
function Notification({ hasNew }: { hasNew: boolean }) {
  return (
    <div>
      <span>알림</span>
      {hasNew && <span className="badge">새 알림</span>}
    </div>
  );
}
```

`hasNew`가 `false`면 `<span className="badge">` 부분은 렌더링되지 않습니다.

**주의: `&&` 앞에 숫자를 쓰지 않습니다.**

```tsx
// 위험: count가 0이면 화면에 "0"이 출력됨
{count && <p>항목 있음</p>}

// 올바름: boolean으로 변환
{count > 0 && <p>항목 있음</p>}
{!!count && <p>항목 있음</p>}   // !!로 boolean 변환도 가능
```

## 패턴 2: 삼항 연산자

두 가지 중 하나를 렌더링할 때 씁니다. `조건 ? A : B` 형태입니다.

```tsx
function LoginStatus({ isLoggedIn }: { isLoggedIn: boolean }) {
  return (
    <div>
      {isLoggedIn ? (
        <button>로그아웃</button>
      ) : (
        <button>로그인</button>
      )}
    </div>
  );
}
```

삼항 연산자 안에 JSX가 길어지면 소괄호로 묶어서 읽기 쉽게 만듭니다.

## 패턴 3: null 반환

아무것도 렌더링하지 않으려면 `null`을 반환합니다.

```tsx
function SuccessBanner({ show }: { show: boolean }) {
  if (!show) return null;

  return (
    <div className="banner">
      저장되었습니다.
    </div>
  );
}
```

컴포넌트 초반에 early return으로 걸러내면, 뒤의 코드가 항상 "정상 상태"를 다룬다는 것이 명확해집니다.

## 패턴 4: 변수로 미리 계산하기

조건이 복잡할 때 JSX 바깥에서 변수로 먼저 계산해두면 JSX가 읽기 쉬워집니다.

```tsx
type FetchStatus = "loading" | "error" | "empty" | "success";

function DataPanel({ status, count }: { status: FetchStatus; count: number }) {
  // JSX 바깥에서 먼저 계산
  let content;

  if (status === "loading") {
    content = <p>불러오는 중...</p>;
  } else if (status === "error") {
    content = <p className="error">오류가 발생했습니다.</p>;
  } else if (status === "empty") {
    content = <p className="muted">항목이 없습니다.</p>;
  } else {
    content = <p>총 {count}개의 항목</p>;
  }

  // JSX는 깔끔하게 유지
  return <div className="panel">{content}</div>;
}
```

## 언제 어느 패턴을 쓸까

| 상황 | 패턴 |
| --- | --- |
| 조건이 참일 때만 보여줌 | `&&` |
| 두 가지 중 하나를 보여줌 | 삼항 연산자 |
| 조건에 따라 아무것도 안 보여줌 | `null` 반환 |
| 세 가지 이상의 상태를 다룸 | 변수로 미리 계산 |
| 조건 로직이 복잡함 | 변수로 미리 계산 또는 early return |

## Early Return 패턴

컴포넌트 상단에서 예외 상황을 먼저 처리하면, 코드 아래쪽은 정상 케이스만 다루게 됩니다.

```tsx
type User = {
  name: string;
  isActive: boolean;
};

function UserCard({ user }: { user: User | null }) {
  // 예외 케이스를 먼저 처리
  if (!user) {
    return <p>사용자 정보를 찾을 수 없습니다.</p>;
  }

  if (!user.isActive) {
    return <p>{user.name}님은 비활성화 상태입니다.</p>;
  }

  // 여기서부터는 user가 존재하고 활성 상태임이 확실
  return (
    <article>
      <h3>{user.name}</h3>
      <span>활성</span>
    </article>
  );
}
```

중첩된 조건보다 early return이 대부분 더 읽기 쉽습니다.

## 흔한 실수

**`&&` 앞에 숫자 사용**

```tsx
// 위험 패턴
{items.length && <List items={items} />}  // items.length가 0이면 "0" 출력

// 올바름
{items.length > 0 && <List items={items} />}
```

**삼항 연산자 중첩**

```tsx
// 읽기 어려움
{isLoading ? <Spinner /> : isEmpty ? <Empty /> : <List />}

// 변수로 분리하면 읽기 쉬움
let content;
if (isLoading) content = <Spinner />;
else if (isEmpty) content = <Empty />;
else content = <List />;

return <div>{content}</div>;
```
