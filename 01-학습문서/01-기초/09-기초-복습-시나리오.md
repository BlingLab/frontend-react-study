# 기초 복습 시나리오

기초 단계의 개념을 실제 코드로 읽고 설명할 수 있는지 확인합니다. 코드를 눈으로 읽으면서 "이 코드는 무엇을 하는가, 왜 이렇게 구성됐는가"를 말로 설명해봅니다.

## 시나리오 1. 카드 하나 만들기

먼저 사용자 한 명을 보여주는 카드를 만듭니다.

```tsx
type User = {
  id: string;
  name: string;
  role: string;
};

function UserCard({ user }: { user: User }) {
  return (
    <article>
      <h3>{user.name}</h3>
      <p>{user.role}</p>
    </article>
  );
}
```

이 코드를 설명할 때는 이렇게 말할 수 있어야 합니다.

- `User` 타입은 데이터 모양을 설명합니다.
- `UserCard`는 `user` 하나를 props로 받습니다.
- props로 받은 데이터를 JSX 안에서 보여줍니다.
- 이 컴포넌트는 아직 state가 필요 없습니다.

## 시나리오 2. 카드 목록 만들기

사용자가 여러 명이면 배열을 `map`으로 렌더링합니다.

```tsx
const users: User[] = [
  { id: "1", name: "Mina", role: "Frontend" },
  { id: "2", name: "Jun", role: "Designer" },
];

function UserList() {
  return (
    <div>
      {users.map((user) => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}
```

여기서 중요한 설명은 `map`과 `key`입니다.

- `map`은 user 하나를 `UserCard` 하나로 바꿉니다.
- `key`는 React가 목록 항목을 구분하는 기준입니다.
- `key`는 컴포넌트 내부에서 읽는 props가 아니므로, 필요하면 `id`를 별도 prop으로 넘겨야 합니다.

## 시나리오 3. 빈 상태 추가하기

실제 UI는 데이터가 없을 때도 고려해야 합니다.

```tsx
function UserList({ users }: { users: User[] }) {
  if (users.length === 0) {
    return <p>등록된 사용자가 없습니다.</p>;
  }

  return (
    <div>
      {users.map((user) => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}
```

여기서는 early return으로 빈 상태를 먼저 처리합니다. 이렇게 하면 아래쪽 코드는 "사용자가 하나 이상 있는 경우"만 다룹니다.

## 시나리오 4. 조건부 렌더링 추가하기

사용자가 활성 상태인지 비활성 상태인지에 따라 다른 UI를 보여줍니다.

```tsx
type UserWithStatus = User & {
  isActive: boolean;
};

function UserCard({ user }: { user: UserWithStatus }) {
  return (
    <article>
      <h3>{user.name}</h3>
      <p>{user.role}</p>
      {user.isActive ? (
        <span className="badge-active">활성</span>
      ) : (
        <span className="badge-inactive">비활성</span>
      )}
    </article>
  );
}
```

설명할 수 있어야 할 것:
- 삼항 연산자로 두 가지 상태를 표현합니다.
- `isActive`가 `true`이면 활성 배지, `false`이면 비활성 배지를 보여줍니다.
- className도 조건에 따라 다르게 줄 수 있습니다.

## 시나리오 5. 계산값 사용하기

완료 개수나 합계처럼 데이터에서 계산할 수 있는 값은 따로 state로 저장하지 않습니다.

```tsx
type Task = {
  id: string;
  title: string;
  done: boolean;
};

function TaskSummary({ tasks }: { tasks: Task[] }) {
  const totalCount = tasks.length;
  const completedCount = tasks.filter((task) => task.done).length;
  const remainingCount = totalCount - completedCount;

  return (
    <div>
      <p>전체: {totalCount}개</p>
      <p>완료: {completedCount}개</p>
      <p>남은 작업: {remainingCount}개</p>
    </div>
  );
}
```

`totalCount`, `completedCount`, `remainingCount`는 모두 `tasks` 배열에서 계산할 수 있습니다. 별도 변수로 저장하지 않아도 `tasks`가 변하면 자동으로 다시 계산됩니다.

## 시나리오 6. children으로 감싸기

레이아웃 역할의 컴포넌트는 `children`을 받을 수 있습니다.

```tsx
function Panel({ title, children }: { title: string; children: React.ReactNode }) {
  return (
    <section>
      <h2>{title}</h2>
      {children}
    </section>
  );
}

function UserPage({ users }: { users: User[] }) {
  return (
    <Panel title="사용자 목록">
      <UserList users={users} />
    </Panel>
  );
}
```

`Panel`은 내부 내용이 무엇인지 몰라도 됩니다. 제목과 감싸는 구조만 책임집니다. 이런 컴포넌트는 `children`이 자연스럽습니다.

## 시나리오 7. 여러 상태를 한꺼번에 다루기

loading, error, empty, success 네 가지 상태를 가진 목록입니다.

```tsx
type FetchStatus = "loading" | "error" | "empty" | "success";

type UserPageProps = {
  status: FetchStatus;
  users: User[];
  onRetry: () => void;
};

function UserPage({ status, users, onRetry }: UserPageProps) {
  if (status === "loading") {
    return <p>사용자 목록을 불러오는 중...</p>;
  }

  if (status === "error") {
    return (
      <div>
        <p className="error">불러오기에 실패했습니다.</p>
        <button onClick={onRetry}>다시 시도</button>
      </div>
    );
  }

  if (users.length === 0) {
    return <p>등록된 사용자가 없습니다.</p>;
  }

  return (
    <div>
      {users.map((user) => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}
```

설명할 수 있어야 할 것:
- early return으로 예외 상황을 먼저 처리했습니다.
- 마지막 return은 정상 케이스(데이터가 있는 경우)만 다룹니다.
- `onRetry`는 부모에서 내려주는 함수입니다. 자식은 클릭 시 이 함수를 호출합니다.

## 시나리오 8. 컴포넌트 조합으로 화면 완성하기

앞에서 만든 컴포넌트들을 조합해 전체 화면을 만듭니다.

```tsx
const sampleUsers: User[] = [
  { id: "1", name: "Mina", role: "Frontend" },
  { id: "2", name: "Jun", role: "Designer" },
  { id: "3", name: "Sol", role: "Backend" },
];

function App() {
  return (
    <main>
      <Panel title="팀원 현황">
        <TaskSummary
          tasks={[
            { id: "t1", title: "리뷰", done: true },
            { id: "t2", title: "배포", done: false },
          ]}
        />
        <UserList users={sampleUsers} />
      </Panel>
    </main>
  );
}
```

이 코드를 읽을 때 확인할 것:
- 데이터(`sampleUsers`, tasks 배열)가 위에서 아래로 흐릅니다.
- 각 컴포넌트는 자신의 역할에만 집중합니다.
- `Panel`은 제목과 레이아웃을, `TaskSummary`는 요약을, `UserList`는 목록을 담당합니다.

## 복습 순서

기초 단계 복습은 아래 순서로 하면 좋습니다.

1. 데이터 타입을 먼저 적습니다.
2. 항목 하나를 보여주는 컴포넌트를 만듭니다.
3. 배열을 받아 목록으로 렌더링합니다.
4. 빈 상태를 추가합니다.
5. 조건부 렌더링이 필요한 곳에 패턴을 적용합니다.
6. 계산 가능한 값은 state 없이 렌더링 중 계산합니다.
7. 감싸는 컴포넌트가 필요하면 `children`을 사용합니다.
8. 작은 컴포넌트를 조합해 전체 화면을 완성합니다.

이 순서를 외우려는 것이 아니라, React UI를 데이터에서 출발해 조립하는 흐름으로 익히는 것이 목표입니다.

## 생각 흐름으로 점검하기

코드를 읽을 때나 처음 만들 때 이 순서로 생각하면 막히는 지점을 찾기 쉽습니다.

1. **이 화면에 필요한 데이터는 무엇인가?** → 타입 정의
2. **어떤 컴포넌트로 나눌 수 있는가?** → 구조 설계
3. **각 컴포넌트는 무엇을 props로 받는가?** → 인터페이스 설계
4. **데이터가 없을 때(빈 상태), 오류가 있을 때 화면은 어떻게 보이는가?** → 상태 처리
5. **계산 가능한 값을 별도 state로 저장하고 있지는 않은가?** → 단순화
6. **데이터는 위에서 아래로 흐르고 있는가?** → 단방향 데이터 흐름
