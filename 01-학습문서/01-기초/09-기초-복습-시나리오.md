# 기초 복습 시나리오

## 시나리오 1. 카드 하나 만들기

먼저 사용자 한 명을 보여주는 카드를 만듭니다.

```tsx
type User = {
  id: number;
  name: string;
  role: string;
};

function UserCard({ user }: { user: User }) {
  return (
    <article>
      <h3>{user.name}</h3>
      <p>{user.role}</p>
    </article>
  );
}
```

이 코드를 설명할 때는 이렇게 말할 수 있어야 합니다.

- `User` 타입은 데이터 모양을 설명합니다.
- `UserCard`는 `user` 하나를 props로 받습니다.
- props로 받은 데이터를 JSX 안에서 보여줍니다.
- 이 컴포넌트는 아직 state가 필요 없습니다.

## 시나리오 2. 카드 목록 만들기

사용자가 여러 명이면 배열을 `map`으로 렌더링합니다.

```tsx
const users: User[] = [
  { id: 1, name: "Mina", role: "Frontend" },
  { id: 2, name: "Jun", role: "Designer" },
];

function UserList() {
  return (
    <div>
      {users.map((user) => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}
```

여기서 중요한 설명은 `map`과 `key`입니다.

- `map`은 user 하나를 `UserCard` 하나로 바꿉니다.
- `key`는 React가 목록 항목을 구분하는 기준입니다.
- `key`는 컴포넌트 내부에서 읽는 props가 아니므로, 필요하면 `id`를 별도 prop으로 넘겨야 합니다.

## 시나리오 3. 빈 상태 추가하기

실제 UI는 데이터가 없을 때도 고려해야 합니다.

```tsx
function UserList({ users }: { users: User[] }) {
  if (users.length === 0) {
    return <p>등록된 사용자가 없습니다.</p>;
  }

  return (
    <div>
      {users.map((user) => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}
```

여기서는 early return으로 빈 상태를 먼저 처리합니다. 이렇게 하면 아래쪽 코드는 "사용자가 하나 이상 있는 경우"만 다룹니다.

## 시나리오 4. children으로 감싸기

레이아웃 역할의 컴포넌트는 `children`을 받을 수 있습니다.

```tsx
function Panel({ title, children }: { title: string; children: React.ReactNode }) {
  return (
    <section>
      <h2>{title}</h2>
      {children}
    </section>
  );
}

function UserPage({ users }: { users: User[] }) {
  return (
    <Panel title="사용자 목록">
      <UserList users={users} />
    </Panel>
  );
}
```

`Panel`은 내부 내용이 무엇인지 몰라도 됩니다. 제목과 감싸는 구조만 책임집니다. 이런 컴포넌트는 `children`이 자연스럽습니다.

## 시나리오 5. 파일을 보지 않고 다시 만들기

기초 단계 복습은 아래 순서로 하면 좋습니다.

1. 데이터 타입을 먼저 적습니다.
2. 항목 하나를 보여주는 컴포넌트를 만듭니다.
3. 배열을 받아 목록으로 렌더링합니다.
4. 빈 상태를 추가합니다.
5. 감싸는 컴포넌트가 필요하면 `children`을 사용합니다.

이 순서를 외우려는 것이 아니라, React UI를 데이터에서 출발해 조립하는 흐름으로 익히는 것이 목표입니다.
