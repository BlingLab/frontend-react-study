# 데이터에서 UI로 생각하기

## 시작은 데이터다

학습 대시보드를 만든다고 생각해봅니다. 먼저 화면이 아니라 데이터를 봅니다.

```ts
type Lesson = {
  id: number;
  title: string;
  level: "기초" | "초급" | "중급";
  completed: boolean;
};

const lessons: Lesson[] = [
  { id: 1, title: "JSX 문법", level: "기초", completed: true },
  { id: 2, title: "컴포넌트 만들기", level: "기초", completed: false },
  { id: 3, title: "State를 고르는 법", level: "초급", completed: false },
];
```

이 데이터만 보고도 화면에 필요한 요소를 추측할 수 있습니다.

- 문서 제목을 보여줘야 합니다.
- 난이도를 보여줄 수 있습니다.
- 완료 여부에 따라 다른 표시가 필요합니다.
- 여러 문서이므로 목록 렌더링이 필요합니다.

## 작은 컴포넌트부터 만든다

문서 하나를 보여주는 컴포넌트를 먼저 생각합니다.

```tsx
type LessonItemProps = {
  lesson: Lesson;
};

function LessonItem({ lesson }: LessonItemProps) {
  return (
    <li>
      <strong>{lesson.title}</strong>
      <span>{lesson.level}</span>
      {lesson.completed ? <span>완료</span> : <span>진행 전</span>}
    </li>
  );
}
```

이 컴포넌트는 `lesson` 하나를 받아 `li` 하나를 반환합니다. 입력과 출력이 분명합니다.

## 배열은 map으로 UI가 된다

이제 여러 문서를 목록으로 보여줍니다.

```tsx
function LessonList({ lessons }: { lessons: Lesson[] }) {
  return (
    <ul>
      {lessons.map((lesson) => (
        <LessonItem key={lesson.id} lesson={lesson} />
      ))}
    </ul>
  );
}
```

여기서 `map`은 "데이터 하나를 UI 하나로 바꾸는 규칙"입니다.

`key={lesson.id}`는 React가 각 문서를 안정적으로 구분하게 해줍니다. 제목이 바뀔 수 있다면 title보다 id가 더 좋은 key입니다.

## 빈 상태도 UI다

목록이 비어 있을 때도 화면은 의도적으로 보여야 합니다.

```tsx
function LessonList({ lessons }: { lessons: Lesson[] }) {
  if (lessons.length === 0) {
    return <p>아직 등록된 학습 문서가 없습니다.</p>;
  }

  return (
    <ul>
      {lessons.map((lesson) => (
        <LessonItem key={lesson.id} lesson={lesson} />
      ))}
    </ul>
  );
}
```

React에서 빈 상태는 예외가 아니라 중요한 UI 상태입니다.

## 계산 가능한 값은 변수로 둔다

완료 개수를 보여주고 싶다면 데이터를 기준으로 계산합니다.

```tsx
function LessonSummary({ lessons }: { lessons: Lesson[] }) {
  const completedCount = lessons.filter((lesson) => lesson.completed).length;
  const totalCount = lessons.length;

  return (
    <p>
      총 {totalCount}개 중 {completedCount}개 완료
    </p>
  );
}
```

아직 state를 배우지 않았지만, 중요한 감각이 하나 있습니다.

`completedCount`는 `lessons`에서 계산할 수 있습니다. 이런 값은 별도의 데이터로 들고 있기보다 필요할 때 계산하는 편이 단순합니다.

## 전체 화면으로 조립하기

마지막으로 작은 컴포넌트를 조합합니다.

```tsx
function StudyDashboard() {
  return (
    <main>
      <h1>React 학습 현황</h1>
      <LessonSummary lessons={lessons} />
      <LessonList lessons={lessons} />
    </main>
  );
}
```

이제 화면은 아래 흐름으로 읽힙니다.

1. `StudyDashboard`가 전체 구조를 잡습니다.
2. `LessonSummary`가 요약 정보를 보여줍니다.
3. `LessonList`가 배열을 목록으로 바꿉니다.
4. `LessonItem`이 문서 하나를 보여줍니다.

이런 식으로 데이터를 중심에 두고 UI를 나누면, React 코드가 훨씬 덜 복잡하게 느껴집니다.

## 읽으면서 생각할 질문

- 화면에 필요한 데이터 모양을 먼저 말할 수 있는가?
- 데이터 하나를 UI 하나로 바꾸는 컴포넌트가 있는가?
- 배열을 렌더링할 때 안정적인 key를 쓰고 있는가?
- 빈 상태를 의도적으로 보여주고 있는가?
- 계산 가능한 값을 불필요하게 따로 저장하고 있지는 않은가?
