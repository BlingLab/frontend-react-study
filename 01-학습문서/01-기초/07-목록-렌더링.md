# 목록 렌더링

## 배열을 화면 목록으로 바꾸기

배열 데이터를 화면에 보여줄 때는 `map`을 씁니다. 각 호출이 JSX element 하나를 반환합니다.

```tsx
const fruits = ["사과", "바나나", "포도"];

function FruitList() {
  return (
    <ul>
      {fruits.map((fruit) => (
        <li key={fruit}>{fruit}</li>
      ))}
    </ul>
  );
}
```

객체 배열은 각 항목을 컴포넌트로 만들어 렌더링합니다.

```tsx
type Todo = {
  id: number;
  text: string;
  done: boolean;
};

const todos: Todo[] = [
  { id: 1, text: "장보기", done: false },
  { id: 2, text: "운동하기", done: true },
];

function TodoItem({ todo }: { todo: Todo }) {
  return (
    <li style={{ textDecoration: todo.done ? "line-through" : "none" }}>
      {todo.text}
    </li>
  );
}

function TodoList({ todos }: { todos: Todo[] }) {
  return (
    <ul>
      {todos.map((todo) => (
        <TodoItem key={todo.id} todo={todo} />
      ))}
    </ul>
  );
}
```

## key가 필요한 이유

React는 목록을 렌더링할 때 각 항목에 `key`를 요구합니다.

항목이 추가, 삭제, 순서 변경될 때 React가 "이 항목이 저 항목과 같은 항목인가"를 판단해야 합니다. `key`가 없으면 React는 위치를 기준으로 판단하는데, 이 경우 항목이 지워지거나 순서가 바뀌면 잘못된 항목을 재사용하는 버그가 생길 수 있습니다.

`key`를 잘 주면 React가 바뀐 항목만 정확하게 업데이트할 수 있습니다.

## key에 올바른 값 사용하기

**서버에서 받아온 id, 또는 항목을 유일하게 식별하는 값**을 씁니다.

```tsx
// 좋음 — id는 항목과 1:1 대응하고 절대 바뀌지 않음
todos.map((todo) => <TodoItem key={todo.id} todo={todo} />)

// 좋음 — 고유한 값이면 문자열도 괜찮음 (변하지 않는 경우)
categories.map((cat) => <Tag key={cat.slug} label={cat.name} />)
```

**index를 key로 쓰는 것은 피합니다.**

```tsx
// 피해야 함
todos.map((todo, index) => <TodoItem key={index} todo={todo} />)
```

index를 key로 쓰면 항목을 앞에서 삭제하거나 순서를 바꿀 때 React가 엉뚱한 항목을 재사용합니다. 순서가 절대 바뀌지 않는 완전히 정적인 목록이라면 예외적으로 허용됩니다.

## key의 특성

- `key`는 같은 목록(sibling) 안에서만 고유하면 됩니다. 다른 목록에 같은 key를 써도 충돌하지 않습니다.
- `key`는 컴포넌트 내부에서 읽을 수 없습니다. `id`가 필요하면 별도 prop으로 전달해야 합니다.

```tsx
// key와 id를 따로 전달
<TodoItem key={todo.id} id={todo.id} todo={todo} />
```

## 빈 목록 처리

빈 배열도 UI 상태입니다. 아무것도 그리지 않기보다 의도한 메시지를 명확히 보여주는 편이 좋습니다.

```tsx
function TodoList({ todos }: { todos: Todo[] }) {
  if (todos.length === 0) {
    return <p className="muted">할 일이 없습니다. 추가해보세요!</p>;
  }

  return (
    <ul>
      {todos.map((todo) => (
        <TodoItem key={todo.id} todo={todo} />
      ))}
    </ul>
  );
}
```

## 필터링과 함께 쓰기

`filter`로 조건에 맞는 항목만 남긴 뒤 `map`으로 렌더링합니다.

```tsx
function ActiveTodoList({ todos }: { todos: Todo[] }) {
  const activeTodos = todos.filter((todo) => !todo.done);

  if (activeTodos.length === 0) {
    return <p>완료되지 않은 항목이 없습니다.</p>;
  }

  return (
    <ul>
      {activeTodos.map((todo) => (
        <TodoItem key={todo.id} todo={todo} />
      ))}
    </ul>
  );
}
```

필터링된 결과를 별도 변수로 먼저 만들어두면 JSX가 읽기 쉬워집니다.

## Fragment와 함께 쓰기

목록 항목이 여러 element를 반환해야 할 때는 Fragment에 key를 붙입니다.

```tsx
type Definition = {
  id: number;
  term: string;
  description: string;
};

function DefinitionList({ items }: { items: Definition[] }) {
  return (
    <dl>
      {items.map((item) => (
        // 단축 문법 <>는 key를 쓸 수 없으므로 React.Fragment를 명시
        <React.Fragment key={item.id}>
          <dt>{item.term}</dt>
          <dd>{item.description}</dd>
        </React.Fragment>
      ))}
    </dl>
  );
}
```

## 흔한 실수

**key를 안쪽 element에 붙이기**

```tsx
// 오류: key가 li 안쪽의 span에 있음
{todos.map((todo) => (
  <li>
    <span key={todo.id}>{todo.text}</span>
  </li>
))}

// 올바름: key는 map이 직접 반환하는 최상위 element에
{todos.map((todo) => (
  <li key={todo.id}>
    <span>{todo.text}</span>
  </li>
))}
```

**`map` 안에서 `return`을 빠뜨리기**

```tsx
// 오류: return이 없으면 undefined 반환
{todos.map((todo) => {
  <li key={todo.id}>{todo.text}</li>  // 렌더링 안 됨
})}

// 올바름 1: 소괄호로 implicit return
{todos.map((todo) => (
  <li key={todo.id}>{todo.text}</li>
))}

// 올바름 2: 중괄호를 쓰면 return 명시
{todos.map((todo) => {
  return <li key={todo.id}>{todo.text}</li>;
})}
```
