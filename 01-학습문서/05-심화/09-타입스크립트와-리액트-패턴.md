# TypeScript와 React 패턴

TypeScript는 React 컴포넌트를 더 안전하게 만드는 도구입니다. props 타입이 명확하면 잘못된 값이 들어올 때 컴파일 단계에서 잡힙니다. 런타임 오류를 줄이고, IDE 자동완성과 문서 역할도 합니다.

## props 타입 정의

`type`과 `interface` 모두 쓸 수 있습니다. 팀 규칙이 없으면 어느 쪽이든 괜찮지만, 한 프로젝트에서 일관되게 사용합니다.

```tsx
type ButtonProps = {
  label: string;
  variant?: "primary" | "secondary";
  onClick: () => void;
  disabled?: boolean;
};

function Button({ label, variant = "primary", onClick, disabled = false }: ButtonProps) {
  return (
    <button
      onClick={onClick}
      disabled={disabled}
      className={`btn btn-${variant}`}
    >
      {label}
    </button>
  );
}
```

optional prop에는 `?`를 붙이고, 기본값은 구조분해 할당에서 처리합니다.

## Discriminated Union으로 상태 표현하기

`status` 필드로 여러 상태를 나눌 때 discriminated union이 효과적입니다. 각 상태에서 어떤 데이터가 있는지 타입이 좁혀지므로, 불가능한 상태 조합을 컴파일 단계에서 막을 수 있습니다.

```tsx
type FetchState<T> =
  | { status: "idle" }
  | { status: "loading" }
  | { status: "success"; data: T }
  | { status: "error"; message: string };
```

```tsx
function ArticlePage({ state }: { state: FetchState<Article> }) {
  if (state.status === "loading") {
    return <p>불러오는 중...</p>;
  }

  if (state.status === "error") {
    return <p role="alert">{state.message}</p>;
  }

  if (state.status === "idle") {
    return <p>검색어를 입력해주세요.</p>;
  }

  return <ArticleBody article={state.data} />;
}
```

`state.status === "error"` 블록 안에서는 TypeScript가 `state.message`에 접근할 수 있다고 알고, `state.status === "success"` 이후에는 `state.data`에 접근할 수 있다고 알아줍니다.

## Generic 컴포넌트

같은 UI 구조인데 데이터 타입만 다를 때 generic을 씁니다.

```tsx
type SelectProps<T> = {
  items: T[];
  getLabel: (item: T) => string;
  getValue: (item: T) => string;
  value: string;
  onChange: (value: string) => void;
};

function Select<T>({ items, getLabel, getValue, value, onChange }: SelectProps<T>) {
  return (
    <select value={value} onChange={(e) => onChange(e.target.value)}>
      {items.map((item) => (
        <option key={getValue(item)} value={getValue(item)}>
          {getLabel(item)}
        </option>
      ))}
    </select>
  );
}
```

사용할 때 타입 인수를 명시하거나 추론을 활용합니다.

```tsx
<Select<User>
  items={users}
  getLabel={(u) => u.name}
  getValue={(u) => u.id}
  value={selectedId}
  onChange={setSelectedId}
/>
```

## 이벤트 핸들러 타입

React에서 자주 쓰는 이벤트 타입입니다.

| 이벤트 | 타입 |
| --- | --- |
| `onChange` (input) | `React.ChangeEvent<HTMLInputElement>` |
| `onChange` (select) | `React.ChangeEvent<HTMLSelectElement>` |
| `onSubmit` | `React.FormEvent<HTMLFormElement>` |
| `onClick` | `React.MouseEvent<HTMLButtonElement>` |
| `onKeyDown` | `React.KeyboardEvent<HTMLInputElement>` |

handler를 컴포넌트 밖에서 정의하거나 테스트할 때 타입을 명시합니다.

```tsx
function SearchInput({ onSearch }: { onSearch: (keyword: string) => void }) {
  function handleKeyDown(event: React.KeyboardEvent<HTMLInputElement>) {
    if (event.key === "Enter") {
      onSearch(event.currentTarget.value);
    }
  }

  return (
    <input
      type="search"
      onKeyDown={handleKeyDown}
      placeholder="검색어를 입력하세요"
    />
  );
}
```

## useRef 타입

DOM 요소를 가리킬 때 초기값을 `null`로 두고 해당 요소 타입을 generic으로 씁니다.

```tsx
function AutoFocusInput() {
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    inputRef.current?.focus();
  }, []);

  return <input ref={inputRef} />;
}
```

값을 저장하는 용도의 ref는 초기값 타입을 명확히 둡니다.

```tsx
const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

function startTimer() {
  timerRef.current = setTimeout(() => {
    // 처리
  }, 1000);
}

function stopTimer() {
  if (timerRef.current !== null) {
    clearTimeout(timerRef.current);
  }
}
```

## children 타입

`ReactNode`는 JSX, 문자열, 숫자, null, undefined, 배열을 포함합니다. 대부분의 children 타입으로 적합합니다.

```tsx
type PanelProps = {
  title: string;
  children: React.ReactNode;
};

function Panel({ title, children }: PanelProps) {
  return (
    <section>
      <h2>{title}</h2>
      {children}
    </section>
  );
}
```

단일 React 요소만 받으려면 `ReactElement`를 씁니다.

```tsx
type TooltipProps = {
  trigger: React.ReactElement;
  content: string;
};
```

## Context 타입

Context에 null 기본값을 두고, 전용 Hook에서 null 체크를 합니다. Provider 밖에서 사용하는 실수를 개발 중에 바로 잡을 수 있습니다.

```tsx
type AuthContextValue = {
  user: User | null;
  signIn: (email: string, password: string) => Promise<void>;
  signOut: () => void;
};

const AuthContext = createContext<AuthContextValue | null>(null);

export function useAuth(): AuthContextValue {
  const ctx = useContext(AuthContext);
  if (ctx === null) {
    throw new Error("useAuth는 AuthProvider 안에서만 사용할 수 있습니다.");
  }
  return ctx;
}
```

## Custom Hook 반환 타입

반환값이 tuple이면 `as const`로 타입을 좁혀 각 요소의 타입이 정확히 추론되게 합니다.

```tsx
function useToggle(initial = false) {
  const [on, setOn] = useState(initial);
  const toggle = useCallback(() => setOn((v) => !v), []);
  return [on, toggle] as const;
}

const [isOpen, toggleOpen] = useToggle();
// isOpen: boolean
// toggleOpen: () => void
```

`as const` 없이 반환하면 `(boolean | (() => void))[]`로 추론되어 타입이 흐려집니다.

반환값이 객체이면 type alias로 명시합니다.

```tsx
type UseFetchResult<T> = {
  data: T | null;
  status: "idle" | "loading" | "success" | "error";
  error: Error | null;
  refetch: () => void;
};

function useFetch<T>(url: string): UseFetchResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [status, setStatus] = useState<UseFetchResult<T>["status"]>("idle");
  const [error, setError] = useState<Error | null>(null);

  // ...

  return { data, status, error, refetch };
}
```

## 컴포넌트 타입 전달 패턴

컴포넌트 자체를 prop으로 받을 때는 `React.ComponentType`을 씁니다.

```tsx
type ListProps<T> = {
  items: T[];
  renderItem: React.ComponentType<{ item: T }>;
  keyExtractor: (item: T) => string;
};

function List<T>({ items, renderItem: Item, keyExtractor }: ListProps<T>) {
  return (
    <ul>
      {items.map((item) => (
        <li key={keyExtractor(item)}>
          <Item item={item} />
        </li>
      ))}
    </ul>
  );
}
```

## 흔한 타입 실수

### any를 무방비로 쓰기

```tsx
// 나쁜 예: 런타임에서야 오류 발생
function processResponse(data: any) {
  return data.items.map((item: any) => item.title);
}
```

`unknown`을 쓰고 타입 좁히기를 합니다. API 응답은 런타임 검증 라이브러리(zod, valibot 등)를 쓰면 타입과 검증을 함께 관리할 수 있습니다.

### as로 타입 강제 변환 남용

```tsx
const user = response as User;  // 실제 데이터가 User 모양임을 보장하지 않음
```

as 사용이 필요하다면 그 이유를 코드 주변에 설명하거나, 런타임 검증으로 교체합니다.

### useState 초기값 타입 좁히기 실수

```tsx
// status가 string으로 추론됨
const [status, setStatus] = useState("idle");

// 올바른 방법
const [status, setStatus] = useState<"idle" | "loading" | "error">("idle");
```
