# Context를 필요한 만큼 쓰기

Context는 부모가 아래 트리의 컴포넌트들에게 값을 제공하는 방법입니다.

깊은 props 전달을 줄일 수 있지만, 모든 state를 Context에 넣으면 오히려 흐름이 흐려집니다. Context value가 바뀌면 그 값을 읽는 컴포넌트들이 다시 렌더링될 수 있습니다.

Provider 범위는 필요한 만큼만 잡습니다. 앱 전체 설정은 넓어도 괜찮지만, 특정 기능의 상태는 해당 기능 영역 안에 두는 편이 읽기 쉽습니다.

State와 dispatch를 분리한 Context로 제공하면, 읽는 값과 변경 함수의 역할을 더 분명히 할 수 있습니다.

## 읽으면서 생각할 질문

- 이 값을 정말 여러 단계 아래에서 읽는가?
- props로 전달하는 편이 더 명확하지는 않은가?
- Provider 범위가 너무 넓지 않은가?
- Context value가 자주 바뀌는가?
- state와 dispatch를 분리하면 읽기 쉬워지는가?
