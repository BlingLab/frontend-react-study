# State를 분류하고 배치하기

상태 설계는 도구 선택보다 분류가 먼저입니다.

## Local UI State

한 컴포넌트나 가까운 영역에서만 쓰는 값입니다.

- input 값
- 탭 선택
- 모달 열림

## Shared Client State

여러 컴포넌트가 함께 읽고 바꾸는 앱 내부 상태입니다.

- 로그인 사용자
- 테마
- 장바구니

## Server State

서버에서 온 데이터입니다. 캐시, 재요청, stale 여부 같은 문제가 따라옵니다.

- 게시글 목록
- 검색 결과
- 사용자 상세 정보

## 읽으면서 생각할 질문

- 이 상태는 local, shared client, server 중 무엇인가?
- 서버 데이터와 UI 선택 상태를 섞고 있지는 않은가?
- state를 올리면 어떤 컴포넌트가 영향을 받는가?
- feature 폴더 안에 둘 수 있는 상태인가?
- 나중에 서버 상태 도구로 분리할 가능성이 있는가?
