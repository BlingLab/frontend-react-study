# 초급 복습 시나리오

초급을 읽고 나면 작은 Todo 기능을 문서 없이 다시 만들어보는 것이 좋습니다.

목표는 멋진 앱을 만드는 것이 아닙니다. state, event, form, 배열 업데이트, state 위치, 계산값을 한 흐름으로 연결하는 것입니다.

아래 시나리오는 하나씩 확장하면서 완성하도록 순서를 잡았습니다. 코드를 눈으로 읽으면서 "이 코드는 무엇을 하는가, 왜 이렇게 구성됐는가"를 말로 설명해봅니다.

## 시나리오 1. 타입과 기본 state 정의

화면부터 만들기 전에 데이터 타입을 먼저 생각합니다.

```tsx
type Todo = {
  id: string;
  title: string;
  done: boolean;
};

type Filter = "all" | "active" | "done";
```

필요한 state를 최소로 정합니다.

```tsx
function TodoPage() {
  const [todos, setTodos] = useState<Todo[]>([]);
  const [filter, setFilter] = useState<Filter>("all");

  return <div>할 일 앱</div>;
}
```

이 코드를 설명할 수 있어야 합니다.

- `todos`는 사용자가 추가/삭제/수정하는 기준 데이터입니다.
- `filter`는 사용자가 선택한 탭 상태입니다.
- 완료 개수, 보이는 목록은 state가 아닙니다. 렌더링 중 계산합니다.

## 시나리오 2. 추가 form 만들기

입력값 state와 submit 처리를 합니다.

```tsx
import { useState, FormEvent } from "react";

function TodoForm({ onAdd }: { onAdd: (title: string) => void }) {
  const [title, setTitle] = useState("");

  function handleSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault();

    const trimmedTitle = title.trim();
    if (trimmedTitle.length === 0) return;

    onAdd(trimmedTitle);
    setTitle("");
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={title}
        onChange={(e) => setTitle(e.target.value)}
        placeholder="할 일을 입력하세요"
      />
      <button type="submit">추가</button>
    </form>
  );
}
```

설명할 수 있어야 할 것:

- `title`은 `TodoForm` 안에만 필요하므로 이 컴포넌트 안에 state를 둡니다.
- `onAdd`는 부모가 내려준 handler입니다. 자식은 추가가 일어났다고 알리기만 합니다.
- `event.preventDefault()`가 없으면 form submit 시 페이지가 새로고침됩니다.
- `title.trim()`으로 앞뒤 공백을 제거합니다.
- 추가 후 `setTitle("")`으로 입력창을 비웁니다.

## 시나리오 3. 배열 state 업데이트

부모에서 추가, 토글, 삭제를 구현합니다.

```tsx
function TodoPage() {
  const [todos, setTodos] = useState<Todo[]>([]);

  function handleAdd(title: string) {
    setTodos((prev) => [
      ...prev,
      { id: crypto.randomUUID(), title, done: false },
    ]);
  }

  function handleToggle(id: string) {
    setTodos((prev) =>
      prev.map((todo) =>
        todo.id === id ? { ...todo, done: !todo.done } : todo,
      ),
    );
  }

  function handleDelete(id: string) {
    setTodos((prev) => prev.filter((todo) => todo.id !== id));
  }

  return (
    <>
      <TodoForm onAdd={handleAdd} />
      {/* TodoList는 다음 시나리오에서 */}
    </>
  );
}
```

설명할 수 있어야 할 것:

- `push`를 쓰지 않고 `[...prev, newTodo]`로 새 배열을 만드는 이유
- `splice`를 쓰지 않고 `filter`로 삭제하는 이유
- 수정할 때 `map`으로 대상 항목만 새 객체로 만드는 흐름
- `(prev) =>` updater function을 쓰는 이유

## 시나리오 4. 목록 렌더링과 컴포넌트 분리

`TodoList`와 `TodoItem`을 만듭니다.

```tsx
type TodoItemProps = {
  todo: Todo;
  onToggle: (id: string) => void;
  onDelete: (id: string) => void;
};

function TodoItem({ todo, onToggle, onDelete }: TodoItemProps) {
  return (
    <li>
      <button onClick={() => onToggle(todo.id)}>
        {todo.done ? "✓ 완료" : "진행 중"}
      </button>
      <span style={{ textDecoration: todo.done ? "line-through" : "none" }}>
        {todo.title}
      </span>
      <button onClick={() => onDelete(todo.id)}>삭제</button>
    </li>
  );
}

type TodoListProps = {
  todos: Todo[];
  onToggle: (id: string) => void;
  onDelete: (id: string) => void;
};

function TodoList({ todos, onToggle, onDelete }: TodoListProps) {
  if (todos.length === 0) {
    return <p>할 일이 없습니다. 추가해보세요!</p>;
  }

  return (
    <ul>
      {todos.map((todo) => (
        <TodoItem key={todo.id} todo={todo} onToggle={onToggle} onDelete={onDelete} />
      ))}
    </ul>
  );
}
```

설명할 수 있어야 할 것:

- `TodoItem`이 `setTodos`를 직접 부르지 않는 이유
- `key={todo.id}`를 쓰는 이유. index를 쓰면 안 되는 상황
- 빈 상태를 early return으로 처리하는 패턴
- `onToggle`과 `onDelete`의 타입 정의 방식

## 시나리오 5. 계산값으로 요약과 필터 추가

계산값과 필터 state를 연결합니다.

```tsx
function TodoPage() {
  const [todos, setTodos] = useState<Todo[]>([]);
  const [filter, setFilter] = useState<Filter>("all");

  // handleAdd, handleToggle, handleDelete는 시나리오 3과 동일

  // 계산값 — state가 아님
  const completedCount = todos.filter((t) => t.done).length;
  const activeCount = todos.length - completedCount;

  const visibleTodos = todos.filter((todo) => {
    if (filter === "active") return !todo.done;
    if (filter === "done") return todo.done;
    return true;
  });

  return (
    <>
      <TodoForm onAdd={handleAdd} />
      <p>전체 {todos.length}개 / 완료 {completedCount}개 / 남은 {activeCount}개</p>
      <div>
        <button onClick={() => setFilter("all")}>전체</button>
        <button onClick={() => setFilter("active")}>진행 중</button>
        <button onClick={() => setFilter("done")}>완료</button>
      </div>
      <TodoList todos={visibleTodos} onToggle={handleToggle} onDelete={handleDelete} />
    </>
  );
}
```

설명할 수 있어야 할 것:

- `completedCount`, `activeCount`, `visibleTodos`가 state가 아닌 이유
- `todos`와 `filter` 두 state가 있을 때 `visibleTodos`가 자동으로 업데이트되는 원리
- 개수를 따로 state로 저장하면 어떤 문제가 생기는가

## 시나리오 6. state를 끌어올리는 상황

검색 기능을 추가하면서 state 위치를 생각해봅니다.

```tsx
type SearchInputProps = {
  keyword: string;
  onKeywordChange: (keyword: string) => void;
};

function SearchInput({ keyword, onKeywordChange }: SearchInputProps) {
  return (
    <input
      value={keyword}
      onChange={(e) => onKeywordChange(e.target.value)}
      placeholder="제목 검색"
    />
  );
}

function TodoPage() {
  const [todos, setTodos] = useState<Todo[]>([]);
  const [filter, setFilter] = useState<Filter>("all");
  const [keyword, setKeyword] = useState(""); // SearchInput과 TodoList 모두 필요

  const visibleTodos = todos
    .filter((todo) => {
      if (filter === "active") return !todo.done;
      if (filter === "done") return todo.done;
      return true;
    })
    .filter((todo) =>
      todo.title.toLowerCase().includes(keyword.toLowerCase()),
    );

  return (
    <>
      <SearchInput keyword={keyword} onKeywordChange={setKeyword} />
      {/* 필터, 목록 등 */}
      <TodoList todos={visibleTodos} onToggle={handleToggle} onDelete={handleDelete} />
    </>
  );
}
```

설명할 수 있어야 할 것:

- `keyword`를 `SearchInput` 안에 두지 않고 `TodoPage`에 두는 이유
- `SearchInput`이 keyword를 props로 받는 구조(controlled)의 장점
- 형제 컴포넌트 간에 데이터를 공유할 때 공통 부모로 state를 올리는 패턴

## 시나리오 7. key로 state 초기화하기

선택된 할 일의 제목을 인라인 편집하는 기능을 만든다고 생각해봅니다.

```tsx
function InlineEditor({
  todo,
  onSave,
  onCancel,
}: {
  todo: Todo;
  onSave: (id: string, nextTitle: string) => void;
  onCancel: () => void;
}) {
  const [draft, setDraft] = useState(todo.title);

  return (
    <input
      value={draft}
      onChange={(e) => setDraft(e.target.value)}
      onKeyDown={(e) => {
        if (e.key === "Enter") onSave(todo.id, draft);
        if (e.key === "Escape") onCancel();
      }}
      autoFocus
    />
  );
}
```

부모에서 `selectedTodoId`가 바뀔 때 `InlineEditor`의 `draft` state를 초기화하고 싶습니다.

```tsx
{selectedTodoId && (
  <InlineEditor
    key={selectedTodoId}  // selectedTodoId가 바뀌면 컴포넌트 자체를 새로 시작
    todo={todos.find((t) => t.id === selectedTodoId)!}
    onSave={handleSave}
    onCancel={() => setSelectedTodoId(null)}
  />
)}
```

설명할 수 있어야 할 것:

- `key={selectedTodoId}`가 없으면 다른 할 일을 선택해도 이전에 입력하던 내용이 남는 이유
- `key`가 바뀌면 React가 컴포넌트를 새로 만드는 원리
- `autoFocus`를 쓰는 이유

## 시나리오 8. 전체 화면 조합

앞에서 만든 컴포넌트들을 조합해 전체 화면을 완성합니다.

```tsx
function App() {
  return (
    <main>
      <h1>할 일 목록</h1>
      <TodoPage />
    </main>
  );
}

function TodoPage() {
  const [todos, setTodos] = useState<Todo[]>([]);
  const [filter, setFilter] = useState<Filter>("all");
  const [keyword, setKeyword] = useState("");

  function handleAdd(title: string) {
    setTodos((prev) => [
      ...prev,
      { id: crypto.randomUUID(), title, done: false },
    ]);
  }

  function handleToggle(id: string) {
    setTodos((prev) =>
      prev.map((t) => (t.id === id ? { ...t, done: !t.done } : t)),
    );
  }

  function handleDelete(id: string) {
    setTodos((prev) => prev.filter((t) => t.id !== id));
  }

  const completedCount = todos.filter((t) => t.done).length;

  const visibleTodos = todos
    .filter((todo) => {
      if (filter === "active") return !todo.done;
      if (filter === "done") return todo.done;
      return true;
    })
    .filter((todo) =>
      todo.title.toLowerCase().includes(keyword.toLowerCase()),
    );

  return (
    <>
      <TodoForm onAdd={handleAdd} />
      <SearchInput keyword={keyword} onKeywordChange={setKeyword} />
      <p>
        전체 {todos.length}개 · 완료 {completedCount}개 · 남은{" "}
        {todos.length - completedCount}개
      </p>
      <div>
        {(["all", "active", "done"] as Filter[]).map((f) => (
          <button key={f} onClick={() => setFilter(f)}>
            {f === "all" ? "전체" : f === "active" ? "진행 중" : "완료"}
          </button>
        ))}
      </div>
      <TodoList todos={visibleTodos} onToggle={handleToggle} onDelete={handleDelete} />
    </>
  );
}
```

## 완성 후 스스로 점검하기

아래 문장을 자기 말로 설명해봅니다.

- `title`(form 입력값)은 왜 state인가?
- `completedCount`는 왜 state가 아닌가?
- 추가할 때 왜 `push`를 쓰지 않는가?
- `TodoItem`은 왜 `setTodos`를 직접 부르지 않는가?
- 목록의 key로 왜 `todo.id`를 쓰는가?
- `keyword`가 왜 `SearchInput` 안이 아닌 `TodoPage`에 있는가?

이 질문에 답할 수 있으면 초급의 핵심 흐름을 잡은 것입니다.

## 생각 흐름으로 점검하기

코드를 읽을 때나 처음 만들 때 이 순서로 생각하면 막히는 지점을 찾기 쉽습니다.

1. **이 화면에 필요한 기준 데이터는 무엇인가?** → 최소한의 state 정의
2. **어떤 값을 계산으로 얻을 수 있는가?** → 파생 state 제거
3. **이벤트는 어디서 발생하고, state는 어디에 있어야 하는가?** → state 위치 결정
4. **각 컴포넌트는 무엇을 props로 받고, 어떤 handler를 호출하는가?** → 인터페이스 설계
5. **배열을 변경할 때 새 배열을 만들고 있는가?** → 불변성 확인
6. **목록의 key가 항목을 안정적으로 식별하는가?** → key 확인
