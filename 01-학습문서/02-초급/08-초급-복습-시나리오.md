# 초급 복습 시나리오

초급을 읽고 나면 작은 Todo 기능을 문서 없이 다시 만들어보는 것이 좋습니다.

목표는 멋진 앱을 만드는 것이 아닙니다. state, event, form, 배열 업데이트, state 위치, 계산값을 한 흐름으로 연결하는 것입니다.

## 만들 화면

아래 기능을 가진 작은 Todo 화면을 만듭니다.

- 할 일 제목을 입력합니다.
- submit하면 목록에 추가합니다.
- 빈 제목은 추가하지 않습니다.
- 할 일을 완료/진행 중으로 토글합니다.
- 할 일을 삭제합니다.
- 전체, 진행 중, 완료 필터를 전환합니다.
- 전체 개수와 완료 개수를 보여줍니다.

이 정도면 초급 핵심이 거의 모두 들어갑니다.

## 먼저 데이터 모양을 정한다

화면부터 만들기 전에 데이터 타입을 먼저 생각합니다.

```ts
type Todo = {
  id: string;
  title: string;
  done: boolean;
};

type Filter = "all" | "active" | "done";
```

이 타입만 봐도 필요한 UI가 보입니다.

- `title`을 입력받는 form이 필요합니다.
- `done`을 바꾸는 버튼이 필요합니다.
- `Filter`를 바꾸는 탭이나 버튼이 필요합니다.
- `todos` 배열을 목록으로 렌더링해야 합니다.

## state는 최소로 둔다

필요한 state는 많지 않습니다.

```tsx
const [todos, setTodos] = useState<Todo[]>([]);
const [title, setTitle] = useState("");
const [filter, setFilter] = useState<Filter>("all");
```

여기서 `todos`, `title`, `filter`는 사용자의 행동으로 바뀌고 렌더링 사이에 기억해야 합니다.

반면 아래 값들은 계산값입니다.

```tsx
const completedCount = todos.filter((todo) => todo.done).length;

const visibleTodos = todos.filter((todo) => {
  if (filter === "active") return !todo.done;
  if (filter === "done") return todo.done;
  return true;
});
```

완료 개수와 보이는 목록은 state로 저장하지 않습니다.

## 이벤트를 하나씩 만든다

처음부터 모든 기능을 만들지 말고 이벤트를 하나씩 붙입니다.

```tsx
function handleSubmit(event: FormEvent<HTMLFormElement>) {
  event.preventDefault();

  const trimmedTitle = title.trim();

  if (trimmedTitle.length === 0) {
    return;
  }

  setTodos((prev) => [
    ...prev,
    { id: crypto.randomUUID(), title: trimmedTitle, done: false },
  ]);
  setTitle("");
}
```

추가는 `submit`에서 처리합니다.

```tsx
function handleToggle(id: string) {
  setTodos((todos) =>
    todos.map((todo) =>
      todo.id === id ? { ...todo, done: !todo.done } : todo,
    ),
  );
}
```

토글은 `map`으로 처리합니다.

```tsx
function handleDelete(id: string) {
  setTodos((todos) => todos.filter((todo) => todo.id !== id));
}
```

삭제는 `filter`로 처리합니다.

이 세 함수가 자연스럽게 읽히면 초급 배열 state 업데이트는 꽤 많이 잡힌 것입니다.

## 컴포넌트를 나누는 순서

처음에는 `TodoPage` 하나에 전부 작성해도 됩니다.

기능이 돌아가면 아래처럼 나눠봅니다.

```tsx
function TodoPage() {
  return (
    <main>
      <TodoForm title={title} onTitleChange={setTitle} onSubmit={handleSubmit} />
      <TodoSummary total={todos.length} completed={completedCount} />
      <TodoFilter value={filter} onChange={setFilter} />
      <TodoList
        todos={visibleTodos}
        onToggle={handleToggle}
        onDelete={handleDelete}
      />
    </main>
  );
}
```

이 구조에서 state는 `TodoPage`에 있습니다.

자식 컴포넌트들은 값을 props로 받고, 사용자가 한 일을 handler로 알려줍니다.

## 완성 후 다시 읽어볼 지점

완성 후에는 코드를 실행하는 것보다 설명하는 연습이 더 중요합니다.

아래 문장을 자기 말로 설명해봅니다.

- `title`은 왜 state인가?
- `completedCount`는 왜 state가 아닌가?
- 추가할 때 왜 `push`를 쓰지 않는가?
- `TodoItem`은 왜 `setTodos`를 직접 부르지 않는가?
- 목록의 key로 왜 `todo.id`를 쓰는가?

이 질문에 답할 수 있으면 초급의 핵심 흐름을 잡은 것입니다.
