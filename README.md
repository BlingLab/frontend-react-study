# frontend-react-study

React를 단계별로 읽고, 예제로 확인하고, 다시 설명하기 위한 학습 저장소입니다.

이 저장소의 목표는 Hook 이름을 많이 외우는 것이 아닙니다. React로 UI를 만들 때 계속 마주치는 질문에 스스로 답할 수 있게 되는 것입니다.

- 이 값은 props인가, state인가, 계산값인가?
- 이 로직은 render, event handler, Effect 중 어디에 있어야 하는가?
- 컴포넌트는 어떤 책임으로 나눠야 하는가?
- 서버 데이터, loading, error, empty 상태는 어떻게 표현해야 하는가?
- 성능 최적화가 필요한지 어떻게 확인할 것인가?
- 내가 만든 구조를 다른 사람에게 설명할 수 있는가?

문서는 책처럼 읽고, 예제는 직접 고치면서 확인하도록 구성했습니다.

## 바로 시작하기

현재 저장소는 React 19.2.x, TypeScript, Vite 기반입니다.

```bash
pnpm install
pnpm run 예제:전체
```

단계별로 바로 열 수도 있습니다.

```bash
pnpm run 예제:기초
pnpm run 예제:초급
pnpm run 예제:중급
pnpm run 예제:고급
pnpm run 예제:심화
```

빌드 확인:

```bash
pnpm build
```

## 이 저장소의 학습 방식

1. 단계 README에서 이번 단계의 목표를 봅니다.
2. 번호가 붙은 문서를 순서대로 읽습니다.
3. 같은 단계의 예제를 실행하고 화면 상태를 확인합니다.
4. 예제 코드에서 state, props, event handler, Effect, 계산값을 구분합니다.
5. 예제 코드를 조금 바꿔서 결과를 확인합니다.
6. 파일을 보지 않고 같은 기능을 다시 만들어 봅니다.
7. 왜 그렇게 만들었는지 짧게 설명합니다.

React를 잘 이해했다는 것은 API 이름을 아는 것이 아니라, 작은 UI 요구사항을 받았을 때 상태, 컴포넌트, Effect, 데이터 흐름을 설명 가능한 구조로 배치할 수 있다는 뜻입니다.
