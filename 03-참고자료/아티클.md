# 아티클

공식 문서로 개념을 잡은 뒤 보조로 읽을 자료를 모읍니다.

지금은 링크를 많이 쌓기보다, 어떤 기준으로 자료를 추가할지 정리해 둡니다. React 생태계는 글이 많고 오래된 정보도 많기 때문에, 아무 글이나 추가하면 오히려 학습 흐름이 흐려집니다.

## 추가 기준

- 공식 문서의 사고방식과 충돌하지 않는 글
- 함수 컴포넌트와 Hook 중심으로 설명하는 글
- "무조건 써라"보다 어떤 상황에서 필요한지 설명하는 글
- 코드 예제가 최근 React와 TypeScript 환경에서 자연스러운 글
- 오래된 class component 중심 글은 특별한 이유가 없으면 제외

## 주제별로 찾을 자료

| 주제 | 찾을 자료의 방향 |
| --- | --- |
| state 위치 | 상태를 올리는 기준과 props 흐름을 설명하는 글 |
| Effect 줄이기 | Effect를 render 계산이나 event handler로 옮기는 리팩터링 사례 |
| custom Hook | UI와 상태 로직을 분리하는 실제 예제 |
| reducer/context | 전역 상태처럼 남용하지 않는 설계 사례 |
| 성능 | Profiler로 측정하고 memoization을 적용하는 사례 |
| 테스트 | Testing Library로 사용자 행동을 검증하는 사례 |
