# 외부자료 리서치

이 문서는 외부 React 학습 자료를 단계별로 분류한 목록입니다. 링크를 많이 모으는 것이 목적이 아니라, 어떤 자료의 관점을 우리 문서에 어떻게 녹일지 정리하는 것이 목적입니다.

## 수집 기준

- 현재 React 공식 문서와 크게 충돌하지 않는 자료를 우선합니다.
- 공개적으로 접근 가능한 문서, 코스, 샘플 PDF, 블로그를 중심으로 둡니다.
- 유료 책 전체 내용이나 저작권이 불명확한 PDF를 그대로 요약하지 않습니다.
- 오래된 class component 중심 자료는 참고만 하고, 본문 예제는 함수 컴포넌트와 Hook 기준으로 다시 씁니다.
- 각 자료는 "읽는 이유"와 "우리 문서에 반영할 점"을 함께 기록합니다.

## 1단계. 기초 자료

### React Learn - Describing the UI

- 링크: https://react.dev/learn/describing-the-ui
- 읽는 이유: JSX, component, props, 조건부 렌더링, 목록 렌더링이 React 입문 흐름의 기준입니다.
- 반영할 점: 기초 단계는 state보다 UI 표현과 props 흐름에 집중합니다.
- 주의할 점: 공식 문서의 예제를 그대로 복사하지 않고, 이 저장소의 카드/목록 예제 문맥으로 바꿉니다.

### MDN React getting started

- 링크: https://developer.mozilla.org/docs/Learn/Tools_and_testing/Client-side_JavaScript_frameworks/React_getting_started
- 읽는 이유: React를 처음 접하는 개발자가 JSX, component, props를 어떤 순서로 만나는지 확인하기 좋습니다.
- 반영할 점: HTML/CSS/JS를 이미 아는 사람에게 React가 추가하는 개념만 분리해 설명합니다.
- 주의할 점: MDN은 교육용 흐름이므로 이 저장소에서는 TypeScript 예제를 함께 유지합니다.

### Full Stack Open Part 1

- 링크: https://fullstackopen.com/en/part1/component_state_event_handlers
- 읽는 이유: props, event handler, state가 작은 예제로 연결되는 흐름이 좋습니다.
- 반영할 점: `onSomething` props와 `handleSomething` handler 이름 규칙을 초급 문서와 예제에 반복합니다.
- 주의할 점: 이 저장소에서는 초반부터 TypeScript 타입을 조금 더 명시합니다.

## 2단계. 초급 자료

### React Learn - Adding Interactivity

- 링크: https://react.dev/learn/adding-interactivity
- 읽는 이유: event handler, state, queueing, object/array update가 초급 핵심과 정확히 맞습니다.
- 반영할 점: state update는 "값을 바꾸는 명령"이 아니라 "다음 렌더링을 예약하는 일"로 설명합니다.
- 주의할 점: 너무 이른 최적화나 전역 상태 도구 이야기를 넣지 않습니다.

### React Learn - Managing State

- 링크: https://react.dev/learn/managing-state
- 읽는 이유: state 구조 선택, state 공유, 보존/초기화 흐름을 단계적으로 다룹니다.
- 반영할 점: 초급에서는 state 위치와 key를 다루고, reducer/context는 고급으로 넘깁니다.
- 주의할 점: 한 번에 reducer/context까지 끌고 오면 초급 학습 흐름이 무거워집니다.

### Kent C. Dodds - Don't Sync State. Derive It!

- 링크: https://kentcdodds.com/blog/dont-sync-state-derive-it
- 읽는 이유: derived state를 별도 state로 동기화하면 왜 문제가 생기는지 설명하는 보조 자료입니다.
- 반영할 점: 초급의 파생 state 문서와 중급의 Effect 제거 문서에 같은 원칙을 다른 난이도로 반복합니다.
- 주의할 점: 블로그의 모든 패턴을 규칙화하지 않고, "중복 state를 줄인다"는 핵심만 반영합니다.

## 3단계. 중급 자료

### React Learn - Escape Hatches

- 링크: https://react.dev/learn/escape-hatches
- 읽는 이유: ref, Effect, custom Hook을 React 바깥 시스템과 연결하는 흐름으로 묶습니다.
- 반영할 점: 중급 단계의 큰 제목을 "외부 시스템과 연결"로 유지합니다.
- 주의할 점: Effect를 데이터 계산용으로 설명하지 않습니다.

### React Learn - You Might Not Need an Effect

- 링크: https://react.dev/learn/you-might-not-need-an-effect
- 읽는 이유: 중급에서 가장 중요한 오해인 "Effect로 state를 맞춘다"를 바로잡습니다.
- 반영할 점: derived value, event handler, key reset으로 Effect를 제거하는 사례를 넣습니다.
- 주의할 점: Effect를 쓰면 안 된다는 뜻이 아니라 외부 시스템 동기화에 집중한다는 뜻으로 설명합니다.

### Overreacted - A Complete Guide to useEffect

- 링크: https://overreacted.io/a-complete-guide-to-useeffect/
- 읽는 이유: Effect dependency를 생명주기 메서드 흉내가 아니라 데이터 흐름으로 이해하는 데 도움이 됩니다.
- 반영할 점: dependency 배열을 속이는 대신 함수 위치, 계산 위치, Effect 분리를 조정합니다.
- 주의할 점: 글이 길고 역사적 맥락이 있으므로 초급 자료로 두지 않습니다.

### Patterns.dev - Hooks Pattern

- 링크: https://www.patterns.dev/react/hooks-pattern
- 읽는 이유: custom Hook을 상태 로직 재사용과 컴포넌트 설계 관점에서 볼 수 있습니다.
- 반영할 점: custom Hook 문서에서 재사용보다 "로직에 이름 붙이기"를 먼저 설명합니다.
- 주의할 점: design pattern을 체크리스트처럼 강제하지 않습니다.

## 4단계. 고급 자료

### React Learn - Scaling Up with Reducer and Context

- 링크: https://react.dev/learn/scaling-up-with-reducer-and-context
- 읽는 이유: reducer와 context를 같이 쓰는 공식 흐름입니다.
- 반영할 점: reducer는 변경 규칙을 모으고, context는 전달 경계를 줄이는 도구로 분리해 설명합니다.
- 주의할 점: context를 전역 store처럼 소개하지 않습니다.

### Kent C. Dodds - Application State Management with React

- 링크: https://kentcdodds.com/blog/application-state-management-with-react
- 읽는 이유: React 자체 도구로 state를 어디까지 관리할 수 있는지 판단하는 데 도움이 됩니다.
- 반영할 점: local state, shared client state, server state를 먼저 분류하고 도구를 고릅니다.
- 주의할 점: 모든 앱이 외부 상태 관리 라이브러리를 피해야 한다는 식으로 단순화하지 않습니다.

### TanStack Query React Docs

- 링크: https://tanstack.com/query/latest/docs/react/overview
- 읽는 이유: server state의 캐싱, 재시도, 재검증, stale 개념을 학습하기 좋습니다.
- 반영할 점: 고급 단계에 server state와 query cache 문서를 둡니다.
- 주의할 점: 처음부터 도구 사용법으로 시작하지 않고, server state가 왜 local state와 다른지 먼저 설명합니다.

### React TypeScript Cheatsheet

- 링크: https://react-typescript-cheatsheet.netlify.app/
- 읽는 이유: props, event, children, reducer action, custom Hook 반환 타입을 실제 TypeScript 스타일로 정리할 때 유용합니다.
- 반영할 점: 각 단계의 예제 타입을 지나치게 어렵지 않게 보강합니다.
- 주의할 점: advanced typing은 고급 이후로 미룹니다.

## 5단계. 심화 자료

### React API Reference - memo, useMemo, useCallback

- 링크: https://react.dev/reference/react/memo
- 읽는 이유: memoization을 언제 적용하고 언제 불필요한지 공식 기준으로 확인합니다.
- 반영할 점: 심화 문서에서 "측정 후 적용" 원칙을 유지합니다.
- 주의할 점: `useMemo`, `useCallback`을 성숙한 코드의 기본 장식처럼 설명하지 않습니다.

### React Compiler Docs

- 링크: https://react.dev/learn/react-compiler/introduction
- 읽는 이유: React Compiler가 memoization 관점을 어떻게 바꾸는지 이해합니다.
- 반영할 점: Compiler가 있어도 state 설계, Effect 설계, 컴포넌트 책임 분리는 여전히 중요하다고 설명합니다.
- 주의할 점: 현재 프로젝트 설정에 Compiler가 적용되어 있다고 가정하지 않습니다.

### React Profiler Reference

- 링크: https://react.dev/reference/react/Profiler
- 읽는 이유: 성능 최적화를 감이 아니라 측정으로 다루기 위한 기준입니다.
- 반영할 점: 심화 성능 문서에 "느린지 먼저 측정한다"는 체크리스트를 둡니다.
- 주의할 점: Profiler API와 DevTools Profiler를 구분합니다.

### React DOM Client APIs

- 링크: https://react.dev/reference/react-dom/client
- 읽는 이유: `createRoot`, `hydrateRoot`, SSR/SSG와 hydration의 차이를 확인할 수 있습니다.
- 반영할 점: 심화와 실전 정리에서 빌드/배포 시 빈 HTML, hydration, framework 선택을 구분합니다.
- 주의할 점: 현재 저장소는 Vite SPA이므로 SSR 구현 문서로 확장하지는 않습니다.

## 6단계. 실전 정리 자료

### Testing Library Guiding Principles

- 링크: https://testing-library.com/docs/guiding-principles/
- 읽는 이유: 테스트를 구현 세부사항이 아니라 사용자 행동 중심으로 작성하는 기준입니다.
- 반영할 점: 테스트 문서에서 state 값을 직접 검증하기보다 화면과 interaction을 검증합니다.
- 주의할 점: 테스트가 모든 구현을 고정하지 않게 합니다.

### user-event Introduction

- 링크: https://testing-library.com/docs/user-event/intro/
- 읽는 이유: click, type 같은 사용자 상호작용을 더 실제에 가깝게 테스트하는 기준입니다.
- 반영할 점: 실전 정리에서 form submit, button disabled, error message를 사용자 행동으로 검증합니다.
- 주의할 점: 단순 DOM event 발생과 사용자 행동 시뮬레이션의 차이를 설명합니다.

### React Testing Library

- 링크: https://testing-library.com/docs/react-testing-library/intro/
- 읽는 이유: React 컴포넌트를 실제 사용자 관점에서 렌더링하고 검증하는 기본 도구입니다.
- 반영할 점: 실전 정리의 테스트 시나리오 문서에서 render, screen, user-event 흐름을 기준으로 둡니다.
- 주의할 점: 컴포넌트 내부 구현 함수 검증을 기본값으로 삼지 않습니다.

- 주의할 점: preview 범위를 넘어선 책 내용은 참고하지 않습니다.

## PDF/책 자료를 다룰 때의 원칙

시중 책 샘플 PDF나 공개 PDF는 목차와 주제 배치를 참고하는 용도로만 사용합니다. 본문 설명과 예제를 그대로 가져오지 않습니다.

참고 가능한 유형:

- 출판사가 공개한 샘플 챕터
- 대학 강의 슬라이드 PDF
- 공식 문서나 공개 코스를 PDF로 묶은 자료
- 라이선스가 명확한 무료 eBook

주의할 유형:

- 출처가 불명확한 전체 책 PDF
- 오래된 class component 중심 책
- React 버전이 오래되어 Hook 이전 패턴을 기본으로 설명하는 자료
- "무조건 Redux", "무조건 memo"처럼 현재 공식 문서 흐름과 어긋나는 자료
