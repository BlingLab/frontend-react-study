# 도구

이 저장소에서 쓰는 도구와, 학습이 진행되면 추가할 만한 도구를 정리합니다.

## 현재 사용하는 도구

| 도구 | 용도 | 공식 문서 |
| --- | --- | --- |
| pnpm | package manager | [pnpm.io](https://pnpm.io/) |
| Vite | 개발 서버와 build 도구 | [vite.dev](https://vite.dev/) |
| React | UI 라이브러리 | [react.dev](https://react.dev/) |
| TypeScript | 정적 타입 검사 | [typescriptlang.org](https://www.typescriptlang.org/) |

## 현재 저장소 명령어

| 명령어 | 용도 |
| --- | --- |
| `pnpm install` | 의존성 설치 |
| `pnpm dev` | 전체 예제 앱 개발 서버 실행 |
| `pnpm build` | TypeScript 검사와 production build |
| `pnpm preview` | build 결과 미리보기 |

## 예제 실행 명령어

| 명령어 | 열리는 예제 |
| --- | --- |
| `pnpm run 예제:기초` | JSX, 컴포넌트, props |
| `pnpm run 예제:초급` | props/state 중심 예제 |
| `pnpm run 예제:중급` | effect/fetching 중심 예제 |
| `pnpm run 예제:고급` | reducer/context 예제 |
| `pnpm run 예제:심화` | performance 예제 |
| `pnpm run 예제:state` | props와 state 예제 |
| `pnpm run 예제:form` | form과 배열 state 예제 |
| `pnpm run 예제:effect` | Effect와 fetching 예제 |
| `pnpm run 예제:hook` | custom Hook 예제 |
| `pnpm run 예제:중급패턴` | 계산값, ref, 필터 UI 예제 |
| `pnpm run 예제:reducer` | reducer와 context 예제 |
| `pnpm run 예제:성능` | 큰 목록과 memoization 예제 |

## 나중에 추가할 만한 도구

| 도구 | 언제 추가할까 | 공식 문서 |
| --- | --- | --- |
| React Router | URL 기반 화면 전환을 다룰 때 | [reactrouter.com](https://reactrouter.com/) |
| Testing Library | 사용자 행동 중심 테스트를 작성할 때 | [testing-library.com](https://testing-library.com/docs/react-testing-library/intro/) |
| user-event | 클릭, 입력 등 실제 사용자 상호작용에 가까운 테스트가 필요할 때 | [user-event](https://testing-library.com/docs/user-event/intro/) |
| Vitest | Vite 프로젝트에서 테스트 runner가 필요할 때 | [vitest.dev](https://vitest.dev/) |
| TanStack Query | 서버 상태, 캐시, 재요청, 동기화를 본격적으로 다룰 때 | [tanstack.com/query](https://tanstack.com/query/latest/docs/react/) |
| MSW | API mock을 브라우저/테스트 환경에서 공통으로 쓰고 싶을 때 | [mswjs.io](https://mswjs.io/) |

## 선택 기준

- 라우팅이 필요해질 때까지 React Router를 먼저 넣지 않습니다.
- 서버 상태가 복잡해지기 전에는 TanStack Query를 미리 넣지 않습니다.
- 테스트는 구현 세부사항보다 사용자가 보는 결과를 검증하는 방향으로 작성합니다.
- 도구를 추가하면 `01-학습문서`와 `02-학습예제`에 왜 추가했는지 같이 기록합니다.
