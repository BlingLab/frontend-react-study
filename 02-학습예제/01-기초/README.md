## 확인할 코드

`src/examples/JsxComponentsExample.tsx`

예제는 세 섹션으로 나뉩니다.

| 섹션 | 다루는 것 |
| --- | --- |
| Props로 카드 목록 만들기 | `map`, `key`, optional props, 빈 상태 처리 |
| 조건부 렌더링 패턴 | `&&`, 삼항 연산자, `null` 반환 |
| children props | 감싸는 컴포넌트, `React.ReactNode` 타입 |

## 예제 읽는 순서

| 순서 | 볼 것 | 확인 질문 |
| --- | --- | --- |
| 1 | `learners` 배열 | UI에 필요한 데이터가 어떤 모양인가? |
| 2 | `ProfileCard` | 데이터 하나가 어떤 JSX 구조로 바뀌는가? |
| 3 | `SkillList` | 배열이 비었을 때도 UI가 있는가? |
| 4 | 조건부 렌더링 섹션 | `&&`, 삼항 연산자, `null` 반환이 각각 언제 쓰이는가? |
| 5 | `Panel`과 `children` | 감싸는 컴포넌트가 내부 내용을 몰라도 되는가? |

## 실습 순서

1. `pnpm run 예제:기초`로 예제를 실행합니다.
2. 브라우저에서 각 섹션을 확인합니다.
3. `learners` 배열에 데이터를 하나 추가합니다.
4. `skills`가 빈 배열인 항목을 하나 더 추가해 빈 상태를 확인합니다.
5. `isNew: true`를 추가해 NEW 뱃지가 표시되는지 확인합니다.
6. `StatusBadge`의 `isOnline`을 `false`로 바꿔봅니다.
7. `NoticeBox`의 `message`를 `null`로 바꿔봅니다.
8. 파일을 닫고 `ProfileCard`를 처음부터 다시 작성해봅니다.

## 확장 과제

| 과제 | 의도 |
| --- | --- |
| learner에 `location` 필드를 추가하고 카드에 표시 | props 타입을 데이터 변경에 맞춰 수정하는 연습 |
| `skills`가 3개 이상이면 "외 N개"로 줄여 표시 | 렌더링 중 계산과 조건부 렌더링 연결 |
| `ProfileCard` 안의 header를 별도 컴포넌트로 분리 | 컴포넌트를 책임 기준으로 나누는 연습 |
| 빈 목록 메시지를 `EmptyState` 컴포넌트로 분리 | 반복될 수 있는 UI 패턴에 이름 붙이기 |

## 스스로 설명해볼 것

- `key`를 `item.id`가 아닌 `index`로 바꿨을 때 왜 위험할 수 있는지 설명합니다.
- `SkillList`를 `ProfileCard`에서 꺼낸 이유를 설명합니다.
- `children`과 일반 props의 차이를 말로 설명합니다.
- `&&` 앞에 숫자를 쓰면 어떤 문제가 생기는지 설명합니다.
- 이 예제에서 state가 필요 없는 이유를 설명합니다.
- 데이터 모양이 바뀌면 어떤 컴포넌트 타입이 같이 바뀌어야 하는지 설명합니다.

## 흔한 실수 포인트

**key 경고 무시하기**: 콘솔에 `Warning: Each child in a list should have a unique "key" prop` 이 뜨면 반드시 수정합니다. 항목 삭제나 순서 변경 시 UI가 잘못 그려집니다.

**className 대신 class 쓰기**: JSX에서는 HTML의 `class` 속성을 `className`으로 씁니다. `class`를 그대로 쓰면 경고가 발생합니다.

**컴포넌트 이름 소문자 쓰기**: `<profileCard />` 처럼 소문자로 시작하면 React가 HTML 태그로 인식합니다. 항상 대문자로 시작합니다.

**`&&` 앞에 숫자**: `{count && <p>...</p>}` 에서 `count`가 `0`이면 화면에 `0`이 출력됩니다. `{count > 0 && <p>...</p>}` 처럼 boolean 조건을 씁니다.
